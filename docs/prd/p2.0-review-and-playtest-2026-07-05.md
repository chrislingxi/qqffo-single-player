# P2.0 规范审计与本地玩家试玩反馈

时间：2026-07-05

## 1. 规范审计反馈

审计范围：

- `docs/prd/mobile-gameplay-standards.md`
- `docs/prd/p1-p3-prd.md`
- `docs/prd/p2.0-rebuild-blueprint.md`
- `data/design/quests.json`
- `data/design/skills.json`
- `data/design/equipment.json`
- `data/design/pets.json`
- `src/app.js`
- `test-results/p2-readiness-audit.md`

结论：

- 本轮前最大问题是任务体系无法证明 30 分钟牵引，且加点、装备、宠物、日常没有进入任务教学链路。
- 本轮已修复任务牵引底座：任务从 17 条扩展到 52 条，主线 24、支线 12、职业 10、循环 6。
- 本轮已接入玩法教学目标：`allocate_attr`、`equip_item`、`enhance_item`、`socket_item`、`train_pet`、`use_pet_skill_pill`、`activate_pet_circle`、`clear_activity`、`learn_skill`。
- `npm run audit:p2` 从 `not_ready` 降为 `risk`，critical 从 3 个降为 0 个。

仍未通过：

- 五职业技能树仍不足：当前每职业 3 主动 + 1 被动，低于 P2 每职业 10-14 主动 + 4-6 被动。
- 装备数据缺少系统溯源字段，虽有可玩装备曲线，但不能宣称原版数值对齐。
- 宠物运行时仍只有 5 只，远低于 P2 20-30 只目标；全宠物目录不能直接算战斗宠物完成。
- 怪物战斗数值仍多为推断，刷怪节奏暂时是单机平衡，不是原版完整还原。

## 2. 模拟自由幻想玩家试玩反馈

模拟前提：只基于本地自由幻想资料和当前游戏，不引入外部记忆。

正向反馈：

- 玩家开局现在更像“有人带着走”：打粉粉兔、收兔毛、收野猪牙、去龙城、加点、换武器、抓宠、培育、进花妖巢穴，链路明显比旧版顺。
- 中期开始出现研究入口：武器加工、打孔、宠物技能丸、宠物法阵、试炼、猎杀令、宝塔、灵泽殿都被任务串起来，不再只是面板里孤立按钮。
- 五职业机器玩家都能完成一条相似成长线，至少不会出现某职业因任务结构卡死。

负向反馈：

- 任务体验还是“骨架成立、血肉不足”：NPC 对话、任务文案、地图事件、任务前后因果还不够像自由幻想。
- 战斗仍偏样机：技能释放、暴击、闪避、死亡、掉落的差异化反馈不足，距离商业手游体感还有距离。
- 美术和 UI 质感仍不稳定：角色和部分 Boss 已明显提升，但地图、面板、技能图标、按钮质感还没有统一商业化。
- 玩法研究目前是“被任务触达”，还不是“玩家真的需要比较方案”。后续需要裸推失败/研究后通过的门槛模拟。

## 3. 本轮直接修复

- 扩展任务表：`data/design/quests.json` 从 17 条扩到 52 条。
- 新增任务目标类型并接入游戏运行：`src/app.js`。
- 新增任务对加点、装备、加工、打孔、宠物培育、宠物技能丸、宠物法阵、日常的牵引。
- 修正战斗绘制顺序：先绘制怪物，再绘制玩家，缓解接敌时怪物盖住角色的问题。
- 增强机器玩家模拟：`tests/player-simulation.mjs` 能跑玩法教学目标，并检查研究动作和日常完成。

## 4. 验收结果

- `npm run test:smoke`：通过。
- `npm run test:player`：通过。五职业均达到 55 级，完成 39 个任务、5 次副本、4 次日常、7 次玩法研究动作。
- `npm run test:browser`：通过。844x390 横屏视口正常，HUD、任务追踪、小地图、技能区、战斗反馈存在。
- `npm run audit:p2`：仍为 `risk`。当前不是 P2.0 完成，只是 P2.0 任务牵引底座显著推进。

## 5. 下一轮建议

1. 优先补五职业技能树数量和字段，解决当前 5 个 high 风险。
2. 建立 `p2-decision-gate-simulation`，验证裸推与研究后的副本通过率差异。
3. 重构技能按钮、冷却遮罩、技能图标和战斗特效，让“能玩”开始接近“像手游”。
