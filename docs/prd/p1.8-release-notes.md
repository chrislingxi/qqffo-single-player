# P1.8 手游样式对标版落地说明

更新时间：2026-07-04

## 当前结论

复盘后，旧 P1.8 只能算“横屏 HUD 方向验证”，不能算工作室级可交付版本。P1.8 重构必须以 `docs/prd/mobile-gameplay-standards.md` 为硬性验收标准。

## 目标

P1.8 不扩玩法系统，专注把 P1.5 的“可玩 H5”改造成横屏 iPhone PWA 单机手游体验。对标重点：横屏场景、2.5D 视角、克制且不堆叠的 HUD、任务追踪、小地图、游戏内浮层、战斗可感知、角色/怪物/宠物 Q 版资产管线。

## P1.8 重构准入标准

- 横屏唯一体验：竖屏只提示横屏游玩。
- PWA 终态：以 iPhone 添加到桌面后的全屏体验验收。
- 单机原则：移除伪玩家、好友、空间、排行等社交/竞争填充。
- HUD 不堆叠：顶部入口数量受控，小地图、角色信息、任务栏不互压。
- 浮层不暂停：背包/地图/宠物/角色打开后底层战斗继续，浮层不遮挡右下技能和核心战斗区。
- 战斗可感知：选怪、靠近、攻击、飘字、死亡、掉落至少有基础表现。
- Q 版资产：符号人和临时几何图形只能标 `prototype_asset`，不能当正式资产。

## 已落地方向

- 横屏全屏壳：`mmo-shell` + `mmo-stage`，画布覆盖主视口。
- 顶部 HUD：头像、等级、职业、血蓝经验条、克制数量的系统入口和红点。
- 左侧任务：主线任务追踪、自动寻路/自动战斗/自动补给开关。
- 右侧 HUD：圆形小地图、单机系统入口、右下普攻与技能轮盘。
- 底部 HUD：战斗/任务日志条。
- 抽屉面板：任务、角色、背包、宠物、地图改为游戏内浮层。
- 2.5D 场景：低饱和石板地面、斜向透视纹理、路径、前景建筑、NPC、怪物、宠物、角色名牌、血条和光点。
- 战斗反馈：锁定怪物、攻击/受击/宠物攻击/击杀/掉落飘字。
- 验收补强：浏览器烟测校验无社交/伪玩家文本、竖屏横屏提示、战斗反馈队列、HUD 数量约束。

## 待补到商业手游完成度的部分

- 角色和 NPC 仍是轻量 Q 版 SVG/Canvas 表达，不是完整纸娃娃动作帧。
- 地图是程序化 2.5D 重绘，不是逐张手绘地图或原图参考 Q 化地图。
- 技能按钮和顶部功能图标为统一风格占位，还不是完整图标套装。
- 技能特效、宠物动作、怪物死亡特效、掉落光柱仍需资产管线补齐。
