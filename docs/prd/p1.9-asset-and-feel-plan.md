# P1.9 高精细资产与手感版计划

更新时间：2026-07-04

## 用户确认边界

- 画质目标：B 档，直接冲参考图的高精细端游质感。
- 首个上线形态：本地 iPhone PWA 添加到主屏幕启动；浏览器直开保留为调试入口。
- 资产策略：允许 AI 生成，基于原版本地化参考做二创和 Q 版化。
- 版权边界：参考资料只用于风格拆解和原创化设计，运行包不直接复用端游原素材。

## 版本目标

P1.9 不扩 P2/P3 系统，优先解决 P1.8 的商业手游完成度短板：角色、宠物、地图、技能、掉落、UI 图标和面板皮肤。

目标体验：

- 进入游戏后第一眼不再像程序化 Canvas 原型，而像一款横屏 2.5D MMO 手游。
- 战斗中有明确的技能释放、命中、掉落和升级反馈。
- iPhone 浏览器横屏下主界面、按钮、文字、技能轮盘不遮挡、不溢出。

## 资产生产优先级

1. 主角五职业：待机、移动、普攻、施法/技能、受击、死亡，每职业先做 4 方向或伪 4 方向。
2. 宠物和 P1 怪物：粉粉兔、小蜗牛怪、食人花、野猪、树妖、鬼火等，先做待机、移动、受击、死亡。
3. 地图分层：桃源村、芳草林、龙城优先，拆成地表、道路、建筑、树木、前景遮挡、交互点。
4. 技能特效：五职业每职业先做 2 个代表性技能，区分近战斩击、枪系突刺、暗器、法术、治疗。
5. 掉落和成长反馈：掉落光柱、装备品质颜色、升级光效、捕捉成功反馈。
6. UI 皮肤和图标：角色、背包、技能、任务、宠物、地图、设置、活动入口、技能按钮。

## 技术落地

- 保持本地 H5 路线，先服务于 iPhone 主屏幕 PWA 横屏启动；Safari/Chrome 浏览器直开作为调试入口。
- 当前 Canvas 可继续承接 P1.9，但资产加载、动画帧、图集 manifest 需要数据化。
- 每个 runtime 资产需要登记来源、生成方式、用途和是否可商用/原创化说明。
- 每次资产替换后跑 smoke/player/browser 验证，并保留截图。

## P1.9 验收标准

- 本地 iPhone 可通过 PWA 添加到主屏幕并横屏游玩，无账号服务器依赖。
- P1.8 已有核心闭环不退化：五职业、任务、自动寻路、自动战斗、掉落、宠物、副本仍可跑通。
- 至少 1 个职业、1 只宠物、3 类怪物、1 张地图完成高精细资产替换并实装动画。
- 至少 5 个技能/战斗反馈从占位效果替换为可见特效。
- 844x390 与主流 iPhone 横屏尺寸下无关键 UI 遮挡。

## 暂不做

- App Store 发布、企业签名、TestFlight 或原生安装包分发。
- 账号系统、服务器、多人、交易、排行榜竞争。
- 直接复制端游客户端资产、原图地图、图标、音乐或文本进运行包。
