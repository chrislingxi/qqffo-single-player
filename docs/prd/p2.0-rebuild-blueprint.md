# P2.0 重建蓝图

更新时间：2026-07-05

## 1. 对齐后的目标

P2.0 不是在当前样机上继续堆系统，而是把当前“能点、能刷”的方向样机重建成个人可用的近商业手游体验。

完成标准：

- 五职业都做，不做单职业标杆后停住。
- 战斗视角采用横屏 2.5D 场景 + 技能按钮，不回到竖屏或网页面板式体验。
- 本地资料优先；本地缺失时允许补采资料，补采来源和置信度必须写入数据字段或缺口表。
- 所有交互达到商业手游基础表达：按钮不堆叠，UI 有图标、有质感、有清晰主次，不乱、表达明确。
- 前期靠任务牵引，玩家能跟任务玩 30 分钟以上；中期靠关卡、等级、副本、加点、技能、装备、宠物等玩法研究继续推动。
- 游戏性、关卡节奏、任务牵引、数值门槛和留存决策类问题必须走 CCGS/游戏策划视角评审。

## 2. 当前关键差距

来自本地审计和 CCGS 旁路评审：

- `data/design/quests.json` 当前只有 17 条任务，主线 9 条、支线 2 条、循环 1 条、职业 5 条，无法支撑 30 分钟任务牵引。
- 任务目标只覆盖 talk/kill/collect/catch_pet/dungeon，没有 learn_skill、allocate_attr、equip_item、enhance_item、train_pet、prepare_dungeon 等玩法教学目标。
- 五职业当前每职业约 4 个技能，低于 P2 每职业 10-14 主动、4-6 被动的目标。
- 装备 15 件、宠物 5 只，离 P2 的 35-80 装备体系和 20-30 宠物目标有明显差距。
- 加点、装备加工、宠物培养等系统是“点击固定收益”，缺少卡点、比较、研究、突破的决策结构。
- 战斗表现仍缺站位分离、Z 排序、技能图标、冷却遮罩、不可用原因、普攻/技能/暴击/闪避/死亡/掉落差异化反馈。

## 3. P2.0 四条建设主线

### 3.1 任务牵引线

目标：玩家能连续跟随任务玩 30 分钟以上，且过程中自然接触核心系统。

必须新增：

- 0-10 级：创建角色、职业确认、普攻/技能、自动寻路、自动战斗、收集、换装。
- 10-20 级：属性加点、技能学习/加点、宠物捕捉、宠物召唤、第一轮小副本准备。
- 20-35 级：转职前置、装备加工、打孔/镶嵌、循环任务、第一批日常入口。
- 35-55 级：灵泽荒原、通天塔、怪物猎杀令、宠物培养、平行世界副本前置。
- 55-80 级：中期副本群、装备升级/拆解/附魂、套装目标、职业进阶技能。

任务目标类型扩展：

- `learn_skill`
- `allocate_attr`
- `equip_item`
- `enhance_item`
- `socket_item`
- `train_pet`
- `use_pet_skill_pill`
- `activate_pet_circle`
- `prepare_dungeon`
- `clear_activity`

### 3.2 玩法研究线

目标：30 分钟后玩家不能只靠自动按钮平推，必须开始做选择。

必须建立三类门槛：

- 属性门槛：不同职业的加点影响命中、暴击、生存或法术效率。
- 装备门槛：裸装或未强化可刷野，但副本首通效率明显不足。
- 宠物/技能门槛：宠物培养、技能等级和技能优先级能影响副本通过率、药耗和耗时。

验收方式：

- 裸装自动推进 vs 合理加点 + 技能 + 强化 + 宠物培养，副本通过率和耗时必须有显著差异。
- 卡点不能只是数值墙；必须给玩家明确提示：缺装备、缺技能、缺宠物、缺防御或缺输出。

### 3.3 数值与资料线

目标：职业、技能、宠物、装备、怪物和副本尽量回连原版资料，不用临时手感数值冒充对标。

优先补齐：

- 五职业 P2 技能树：每职业至少 10 个主动、4 个被动，字段包含等级、消耗、冷却、倍率、目标、效果、来源和缺口说明。
- 35-80 装备等级段：1/12/28/38/50/65/80 档，覆盖武器、防具、饰品、套装和职业限制。
- 20-30 只宠物：区分攻宠、防宠、法宠、辅助，带捕捉来源、成长类型、技能、资质或缺口标记。
- 怪物战斗数值：官方资料没有时允许资料站/玩家观察/单机推断，但必须标记 `source_confidence` 或 `stat_confidence`。

### 3.4 美术交互线

目标：横屏第一眼不 low，战斗过程能看懂，按钮和面板不再乱。

必须重构：

- HUD：角色状态、任务追踪、小地图、系统入口、技能区、自动开关、聊天/日志固定层级。
- 技能按钮：图标、冷却遮罩、蓝耗/不可用原因、自动释放状态。
- 自动按钮：自动寻路/自动战斗/自动补给必须有开关状态、当前目标和执行反馈，不能只是文字按钮。
- 战斗层级：角色、怪物、宠物、掉落、目标圈、技能特效、飘字分层绘制，接敌后不互相遮挡到不可读。
- 特效：普攻、职业技能、暴击、闪避、受击、死亡、掉落必须有不同表现。

## 4. 施工顺序

### Phase A：验收闸门先行

- 新增 `audit:p2`，每轮输出 P2.0 readiness。
- 新增 30 分钟任务链路模拟。
- 新增 UI 重叠检测。
- 新增战斗可读性检测。

### Phase B：任务和玩法数据

- 扩充 `quests.json` 至至少 45 条。
- 扩充任务目标类型并接入 `guideQuest()`。
- 补 P2 技能、装备、宠物的最小量级。
- 把所有新增数据带上来源或缺口标记。

### Phase C：战斗 UI 和交互重构

- 重排 HUD，清理堆叠入口。
- 技能区图标化和冷却化。
- 自动战斗状态机可视化。
- 战斗对象分层、站位和特效分层。

### Phase D：卡点与研究循环

- 设计 2-3 个 P2 中期门槛：转职、副本首通、通天塔/猎杀令。
- 建立裸推失败、研究后通过的模拟。
- 调整任务文案和提示，让玩家知道该研究什么。

## 5. 当前闸门

运行：

```bash
npm run audit:p2
```

- `test-results/p2-readiness-audit.json`
