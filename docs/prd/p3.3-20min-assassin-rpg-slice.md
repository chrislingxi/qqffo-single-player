# P3.3 20分钟刺客RPG体验纵切 PRD

## 目标

P3.3 不再以单张效果图或静态切片作为完成标准，而是交付一条可在 iPhone PWA 横屏游玩的 20 分钟刺客体验线。画面观感以此前确认的商业化 RPG 界面为目标，产品逻辑充分使用本地化的 QQ自由幻想端游资料，缺失信息只做项目内原创补齐，不直接复用原始美术。

## 核心体验

- 开局固定刺客，进入桃源村后自动开启主线指引、自动寻路、自动战斗、自动补给。
- 主线覆盖 Lv.1-20 的成长节奏，包含对话、击杀、收集、属性加点、装备穿戴、宠物捕捉、副本、日常试炼。
- 所有任务目标支持跨地图传送与目标寻路，避免地图切换后不知道去哪里。
- 野外怪物是地图生态常驻 spawn，不是玩家身边临时刷一只；怪物有分布、游荡、复活时间和精英/BOSS区域。
- 宠物捕捉在 P3.3 纵切中必须可达，不能被概率卡死。

## 地图标准

- 每张地图必须有显著差异：桃源村、芳草林、竹水秘径、阳原平原、花妖巢穴。
- 地图画面优先采用 AI 二创 Q 版化商业素材，不走线框/通用底图。
- 地表、地标、氛围、边界均需有表达：石板/草地/泥路/水域/法阵，建筑/竹桥/荷塘/树丛/洞穴，光效/雨/雾/灵光等。
- 小地图同步显示地形、怪物点、玩家点、任务目标点。

## 战斗标准

- 刺客技能必须形成职业感：快速普攻、影袭/控制、淬毒、连击被动。
- 技能特效模板覆盖斩击、冲击、残影、毒雾、宠物爪击、范围提示、精英/BOSS预警。
- 战斗角色和怪物进入统一 Y 轴排序，避免玩家/怪物层级遮挡错误。
- 战斗表现要有受击停顿、击退、屏幕震动、暴击/多段飘字、掉落提示。

## 面板标准

- 顶部系统入口必须是图形 icon，文字只能做 label。
- 必须具备任务、角色/技能、背包/装备、宠物、地图、日常/副本面板。
- 宠物面板保留原版强系统结构：模型/头像、生命法力、等级、信赖、饥渴、基础属性、加点、技能格、法阵、宠物列表。
- 面板打开不应造成游戏完全停滞；但 HUD 不得遮挡主要战斗反馈。

## 验收

- `npm run test:p3-rpg`：检查 P3.3 运行时、地图资产、刷怪生态、任务类型、技能模板、宠物必达和 HUD 样式。
- `npm run test:p3-visual`：保留商业视觉门禁，确保 P3.2 效果图级视觉约束没有丢失。
- `npm run test:ui-experience`：检查 icon、侧边重复按钮、Y 轴排序、面板性能和生态刷怪。
- `npm run verify:vertical-slice`、`npm run test:vertical-player`、`npm run test:player`：验证数值与玩家模拟链路。
- 浏览器实机检查 `?reset=1&p33=1`：横屏首屏、自动任务、地图差异、战斗、宠物、面板都必须可见可用。
