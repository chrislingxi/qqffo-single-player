# P3.2 商业级视觉纵切 100% 对标版

## 目标

P3.2 不以功能覆盖率作为完成标准，而以实际运行截图的第一眼观感作为完成标准。

完成定义：打开 `?reset=1&visual=1` 后，iPhone 横屏 PWA 的首屏实战画面必须和 `docs/visual-target/p3_2_game_interface_effect_v1.png` 属于同一档商业 2.5D 手游观感，不能再呈现 HTML 原型、线框小游戏或低保真占位感。

## 纵切边界

- 地图：复用本地资料 `fangcao_east / 芳草林东`，展示名为 `竹水秘径`。
- 职业：刺客。
- 等级：Lv.20 展示态。
- 怪物：粉粉兔/步行鸟/食人花等芳草林东生态怪物，首屏锁定食人花战斗。
- 技能：刺客已学习主动技能全部进入技能盘，并保留冷却、消耗、打击反馈。
- 系统：HUD、任务追踪、小地图、顶部系统 icon、技能盘、聊天条、背包/宠物/角色面板入口。

## 画面门禁

- 必须使用高质量地图底图资产 `assets/game/qstyle/p32/bamboo-water-battlefield.png`，不能只靠 canvas 几何绘制。
- 地图需要呈现水域、竹桥、竹林、泥石路、花草、岩石、灯笼、前景遮挡和光效。
- HUD 必须使用图形 icon 和金玉材质框，文字只能作为 label，不能作为 icon 本体。
- 小地图需要地形感、怪物点、玩家箭头和地图名。
- 技能区需要大普攻按钮、四个技能按钮、冷却数字、技能图形和标签。

## 战斗门禁

- 首屏必须处于战斗态。
- 至少包含挥砍弧光、命中爆点、地面法阵、暴击飘字、受击闪白、击退、屏幕震动。
- 怪物不能全部贴在玩家身边，必须保持地图生态分布。
- 玩家和怪物层级按地图 y 坐标排序，不能出现人物/怪物遮挡错误。

## 验收方式

每次交付必须提供：

- 效果图：`docs/visual-target/p3_2_game_interface_effect_v1.png`
- 实际运行截图：`test-results/p3_2_runtime_showcase.png`
- 对比结论：地图、UI、怪物、技能动效、横屏体验五项逐条说明
- 自动测试：`npm run test:p3-visual`、`npm run test:ui-experience`、浏览器截图检查

若实际截图与效果图一眼不同档，不能宣布 P3.2 完成。
