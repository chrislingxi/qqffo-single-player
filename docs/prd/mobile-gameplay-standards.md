# 横屏 iPhone PWA 手游体验硬性规范

更新时间：2026-07-04

## 适用范围

本规范适用于 QQ自由幻想单机版 P1.8 及后续全部移动端版本。未满足本规范的版本只能标记为“方向验证/原型”，不能标记为“完成/上线”。

## 1. 终态与方向

- 游戏体验只按横屏建设。
- iPhone 竖屏只显示“请横屏游玩”，不做竖屏游戏布局。
- 终态按 iPhone 添加到桌面后的 PWA 全屏体验验收；Safari 地址栏直开只作为调试入口。
- 所有 HUD 必须避开刘海、Home Indicator、系统手势区和 PWA safe area。

## 2. HUD 与层级

- HUD 必须有固定层级：角色状态、任务追踪、小地图、战斗操作、系统入口、聊天/日志、浮层面板。
- 禁止 icon 堆叠：顶部入口数量、尺寸、间距、红点位置必须有上限，不能互相压字、压小地图、压角色状态。
- 系统入口 icon 不能用纯文字或汉字圆牌冒充图标；任务、背包、地图、宠物、日常、后期、技能、补给等高频入口必须有可辨认的图形符号、按下态、激活态和红点位置规范。
- 所有交互控件必须达到商业手游基础表达标准：按钮不堆叠、不压字、不遮挡关键画面；必须有图标、质感、状态反馈和清晰主次，不能只用生硬文字按钮完成交互。
- 自动寻路、自动战斗、自动补给、攻击、技能、背包、宠物、地图等高频交互必须表达明确：玩家一眼能看懂当前开关状态、执行目标、冷却/不可用原因和下一步可操作项。
- 默认首屏不得出现大面积功能面板。
- 背包、地图、宠物、角色等功能面板必须是游戏内浮层，打开后底层游戏继续运行。
- 浮层不得遮挡核心战斗区、右下技能区、玩家/怪物交互区；必须有明确关闭方式。
- 所有核心面板都必须按正式系统设计，不允许只用临时卡片列表承载：任务、角色属性、技能、背包/装备、地图、宠物、日常、后期养成都要有标题区、状态摘要、主体信息区、操作区和关闭方式。

## 3. 单机原则

- 本项目是本地 iPhone 单机刷爽版，不设计伪玩家、玩家社交、玩家交易、排行榜竞争、好友社交等 MMO 填充。
- 画面密度只能由 NPC、怪物、宠物、场景物件、掉落、技能特效、任务反馈补足。
- 任何“其他玩家”表现都视为偏离目标，必须移除。

## 4. 战斗交互

- 打怪必须可感知：选怪、寻路靠近、普攻动作、技能反馈、冷却、伤害飘字、受击、怪物死亡、掉落反馈至少具备基础表现。
- 自动战斗必须有状态反馈，玩家能看懂当前正在追踪/接敌/攻击/拾取。
- 玩家、怪物、宠物、掉落和技能特效必须有稳定层级：角色接敌后不能被怪物遮住，目标血条、受击、暴击、技能命中特效不能互相覆盖到不可读。
- 普攻、主动技能、暴击、闪避、死亡、掉落必须有不同反馈；同一套闪光/圆圈不能冒充全部战斗表现。
- 打开背包、地图、宠物等浮层时，自动战斗不暂停，但玩家仍能看见战斗反馈。
- 怪物不能只作为单点战斗入口存在；野外地图必须有多只怪物分布在地图各处，具备刷新时间、刷新半径、死亡消失和复活表现。
- 每张地图必须有个性化地表、地标、氛围或天气，不允许所有地图共用一张底图或同一套场景道具。

## 4.1 任务与玩法牵引

- P2 及后续版本必须把任务体系作为前期 30 分钟的主牵引：玩家能连续跟随主线、支线、职业、收集、循环和副本前置任务推进，不依赖自己猜下一步做什么。
- 任务不能只是奖励弹窗，必须串起地图、NPC、怪物、掉落、技能学习、装备穿戴、宠物培养、副本解锁等核心玩法。
- 前期靠任务牵引，中期靠关卡、等级、副本和养成门槛牵引。约 30 分钟后，玩家需要开始研究属性加点、技能优先级、装备强化、宠物培养和副本策略，才能继续顺利推进。
- 每个阶段都必须有明确“为什么继续玩”的目标：升级、转职、解锁技能、换装、强化、宠物成长、副本首通、日常资源或高阶养成。
- 游戏性、关卡节奏、数值门槛、任务牵引和留存决策类问题必须引入 CCGS/游戏策划视角评审；若 CCGS 建议与 PRD 冲突，记录冲突点并回到产品决策。

## 5. Q 版资产管线

- 角色、怪物、宠物资产必须按“原版参考 -> Q 化设定 -> 动作/朝向/尺寸规范 -> 游戏内验收”生产。
- 不能用符号人、圆点、纯几何临时 SVG 冒充最终 Q 版资产。
- 暂未完成动作帧时，必须在文档和资产清单中标记为 `prototype_asset`，不能写成正式资产。

## 6. 验收方式

- 每轮交付必须同时通过技术测试和体验验收。
- 技术测试包括数据 smoke、五职业模拟、横屏 PWA/浏览器视口测试、横向溢出检查。
- 体验验收必须使用 iPhone 横屏 PWA 或等价横屏视口截图，检查 HUD 不堆叠、浮层不遮挡、战斗反馈可见、无伪玩家，且所有交互都达到“按钮不堆叠、UI 有图标、有质感、有清晰主次、不乱、表达明确”的标准。
- P2 体验验收必须包含至少 30 分钟任务链路模拟或等价自动化脚本，证明任务体系能牵引玩家进入技能、加点、装备、宠物和副本研究，而不是只完成零散点击。
- 产品、交互、美术、技术、QA 必须分别给出检查结论；只通过脚本不算完成。

## 7. 准入结论

- `方向验证`：能展示大致视觉方向，但未满足本规范。
- `可试玩`：核心链路可玩，主要 HUD/交互可用，但资产或特效仍是原型。
- `可交付`：满足本规范，且缺口被明确记录。
