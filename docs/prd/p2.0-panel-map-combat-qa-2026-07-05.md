# P2.0 面板、地图、战斗体验闭环记录

日期：2026-07-05

## 本轮目标

把 2.0 体验从“能点能刷”推进到“个人可用的 RPG 手游形态”：地图个性化、怪物分布刷新、任务跨图传送、图形化入口、角色/技能/装备/宠物/地图等全量面板，以及战斗层级和技能反馈。

## 已落地

- 新增 `data/design/map_scene_profiles.json`，每张主要地图拥有独立地表、道具、天气和氛围配置。
- 野外怪物改为地图内多实例分布，具备刷新半径、死亡消失、按时间复活和轻微游走。
- 任务指引跨地图时明确执行任务传送，避免玩家到 A 地后任务目标在 B 地找不到路。
- 顶部入口、快捷开关、技能按钮改为图形符号，不再依赖纯汉字圆牌。
- 任务、角色属性/技能、装备背包、宠物、地图等面板改为统一 RPG 窗口骨架：标题区、状态摘要、主体信息区、操作区。
- 宠物面板补齐宠物肖像区、生命/攻击/防御/信赖/饥饿/技能等级、技能格、宠物列表、全宠物图鉴说明。
- 技能面板显示技能节点、未解锁/已学/冷却状态。
- 装备面板显示装备槽、耐久、强化、孔位、加工/打孔/附魂操作。
- 战斗怪物使用地图实例坐标进入战斗，不再永远固定贴在角色身后。
- 技能特效按火、冰、毒、光、枪、范围法阵等模板区分，遭遇怪物的冗余锁定大字改为无文字目标圈，降低中心遮挡。

## QA 测试

- `node --check src/app.js`：通过。
- `npm run test:smoke`：通过，P3 数据烟测通过。
- `npm run test:player`：通过，五职业均可模拟推进到 Lv.55，完成 39 个任务、5 个副本、4 个日常、7 次研究行为、3 次属性分配。
- `npm run test:browser`：通过，真实浏览器横屏 844x390 视口可运行，默认面板不遮挡，面板可滚动，打开面板后底层时间继续推进，P3 面板入口可点击，高阶副本可初始化。

## 模拟自由幻想玩家体验反馈

结论：本轮后从“网页卡片小游戏”推进到“可识别的横屏 RPG 手游框架”。地图、入口、面板和战斗反馈都有了正式结构，玩家能理解任务、传送、刷怪、装备、宠物和技能之间的关系。

仍需后续补齐：

- P2 数据审计仍为 `risk`：五职业技能树深度不足，每职业目前 3 主动 + 1 被动，目标是至少 10 主动 + 4 被动。
- 装备、宠物、怪物数值仍有较多 `inferred`，不能宣称已经完整贴近原版数值。
- 当前 icon 是项目内图形符号，不是最终 AI 高质量成套 icon；视觉质感比纯文字好，但还未达到真正商业手游全套美术资源标准。
- 地图已具备差异化配置，但仍是程序化 Q 化场景，不是逐张高质量 AI 地图成图。

## 本轮验收结论

本轮 UI/交互/地图/怪物/面板/战斗体验闭环通过 QA 与模拟玩家测试；P2 资料与数值完备度缺口继续保留，不包装为已完成。
