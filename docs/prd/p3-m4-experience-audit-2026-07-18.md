# P3.0 M4.1 体验自测与玩家模拟记录

日期：2026-07-18

## 对标口径

- 参考 `data/design/p3_m4_goal.json`：P3.0 M4 是 2.5D 刺客商业纵切，必须优先保证 iPhone 横屏 PWA、地图个性化、生态刷怪、刺客技能、宠物/装备/任务闭环。
- 参考 `docs/prd/mobile-gameplay-standards.md`：默认横屏、不可出现 icon 堆叠、不可用纯文字冒充 icon、单机版剔除社交/交易/玩家竞争、面板不可破坏核心战斗体验。
- 参考本地 QQ 自由幻想资料与截图方向：地图需要有石板/草地/水域/竹桥/地标差异；怪物在地图上可见、分区刷新；战斗需要接近、出手、命中、飘字、死亡/掉落反馈。

## 本轮发现

- 自动测试过关但体验仍低：原测试更多验证数据文件、任务链、P3 功能存在，没有验证视觉层级、HUD 堆叠、素材降级与面板运行成本。
- 右侧模块重复：顶部已有任务/技能/背包/地图/宠物/日常，右侧又生成背包/地图/日常，横屏 iPhone 上形成视觉堆叠。
- 文本按钮破坏质感：聊天栏仍用“战”“存”作为按钮本体，不符合原版 RPG icon 化密集 HUD 的感受。
- 关闭面板仍重绘：默认 HUD 关闭时，`renderGame()` 仍刷新完整面板 HTML，容易带来卡顿和交互不稳定。
- 深度层级不稳定：NPC/普通怪/玩家/战斗怪不是统一按世界 Y 轴绘制，可能出现玩家、怪物互相遮挡关系错误。
- 素材失败降级很廉价：AI 二创素材未加载完或失败时退化成红圆，玩家会直接感知为低级 demo。

## 已修复

- 移除右侧重复模块按钮，保留顶部系统入口与右下战斗技能区。
- 聊天栏状态/保存改为图形 icon 按钮，文字只作为状态描述。
- 默认关闭面板不再持续重建，仅打开抽屉或切换面板时渲染内容。
- 拆分 `drawNavigationAids` 与 `drawWorldActors`，所有 NPC、怪物、战斗敌人、玩家进入统一 Y 轴排序。
- 新增 `drawFallbackSprite`：角色、兔宠、食人花、巨熊/野兽、灵体类素材未加载时用 Q 版轮廓降级，不再显示红圆。
- 新增 `test:ui-experience`，把上述问题写入质量门禁，并接入发布流程。
- 提升 `DATA_VERSION` 与 Service Worker 缓存版本，避免手机继续读取旧包。

## 模拟自由幻想玩家反馈

- 进入游戏后的第一印象应像“端游 HUD 手机化”，而不是网页按钮堆叠；这轮先砍掉重复入口和文字按钮，保留信息密度。
- 刺客刷怪时必须看到地图上的怪，而不是靠近后凭空出现；当前生态刷新仍保留，下一轮重点应继续提升怪物巡游与聚落感。
- 战斗不能固定站桩互相打；当前已有冲刺、命中、飘字、宠物出手，但还需要更多位移、受击顿挫和技能前后摇。
- 面板应该是原版系统面板的手游适配，不应该干扰战斗；这轮降低了关闭面板的成本，下一轮要继续重构装备/技能/宠物面板的视觉资产。
- 美术仍未到商业级：本轮只是保证低级降级不出现，真正达到截图级需要建立角色、怪物、地图地标、技能特效的成套高质量资产生产与验收流程。

## 后续验收重点

- 用浏览器截图和 DOM 几何检查验证 HUD 无重叠。
- 用 7-10 分钟自动刷怪脚本验证任务传送、跨地图寻路、怪物刷新、战斗特效、掉落与升级闭环。
- 用自由幻想玩家视角记录“像不像、爽不爽、是否迷路、是否卡顿、是否低级”五类问题，和自动 QA 一起作为发布前门禁。
