# P2.0 RPG UI 与战斗体验优化记录

时间：2026-07-05

## 1. 外部学习抽象

参考公开资料后，本轮只抽象通用规律，不复用任何外部游戏素材：

- HUD 应该透明覆盖在游戏画面上，让玩家不离开战斗也能读到关键信息。
- 高频按钮需要明确状态：按下态、选中态、冷却、不可用原因、当前目标。
- 技能冷却适合用半透明遮罩/倒计时表达，不能只在日志里提示。
- 战斗反馈要同时具备空间位置和事件差异：谁攻击、命中谁、是否暴击、是否受击、是否掉落。
- 触控游戏界面应保持布局灵活、可读、响应明确；无关或当前不可用的操作应减少屏幕占用。
- 重要信息不能只靠声音或颜色表达，尤其在小屏触控上，交互元素需要足够间距、清晰文本/符号和视觉反馈。

参考来源：

- Apple Human Interface Guidelines: Game controls.
- Game Accessibility Guidelines: Basic / Full list.
- Unity Manual: coroutines and cooldown/timer patterns for gradual feedback.

## 2. 本地自由幻想玩家视角问题

模拟玩家前提：只看本地自由幻想资料和当前游戏。

原问题：

- 首页/HUD icon 更像文字圆按钮，不像商业手游入口。
- 点开背包、地图、宠物、日常时缺少按下态和当前选中反馈。
- 技能按钮没有冷却遮罩，玩家不知道为什么技能没放。
- 人和怪固定站位互相扣血，战斗像数值面板，不像 RPG 战斗。
- 伤害、技能、掉落、日志同时堆在屏幕上，画面噪音过大。

## 3. 本轮修复

- 顶部 icon 改为更明确的“符号 + 标签 + 角标 + 选中态”结构。
- 技能轮盘新增普攻激活态、技能图标、冷却遮罩、倒计时和不可用视觉。
- 战斗新增动作状态：玩家攻击前冲、怪物受击后退、宠物攻击前冲。
- 普攻、技能、暴击、宠物攻击、掉落新增更清晰的差异化特效。
- 浮字数量从 28 收敛到 18，战斗日志从 5 条收敛到 3 条，横屏只显示 2 条。
- 隐藏右侧重复系统入口，给技能轮盘让出空间，避免技能区与系统 icon 堆叠。
- 二次收敛左侧 HUD：任务追踪改为窄追踪条，自动任务/寻路/战斗/补给改为独立胶囊按钮，移除大块黑底遮挡。
- 缓存版本更新为 `p2-experience-01`，避免 PWA 吃旧包。

## 4. 试玩反馈

改善：

- 顶部入口更像手游 HUD，不再只是文字圆按钮。
- 技能区现在能看出冷却和可用状态，普攻按钮有自动战斗状态反馈。
- 战斗中玩家、宠物、怪物不再完全傻站，攻击和受击有微位移。
- 暴击、技能、掉落比上一版更容易识别。
- 右侧技能区不再被侧边入口挤压。
- 左侧任务和自动按钮不再像面板压住战斗区，视觉噪音下降。

仍不足：

- 左侧任务区和自动按钮仍偏重，后续应改为更轻的任务折叠/目标追踪条。
- 技能图标目前是符号化，不是 AI Q 版正式技能图标。
- 地图场景仍是程序化背景，不是原版地图参考后的 Q 化场景。
- 战斗动画仍是“微位移 + 特效”，还没有完整动作帧、朝向、前摇/后摇和连击节奏。
- P2 审计仍为 `risk`，剩余 high 问题主要是五职业技能树深度不足。
