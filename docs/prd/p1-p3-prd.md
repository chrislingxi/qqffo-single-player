# QQ自由幻想端游本地单机化 PRD（P1/P2/P3）

更新时间：2026-07-04
适用目标：本地 iPhone 单人游玩，H5 或端内壳均可
资料基线：`docs/feature/feature.md` 与 `docs/reference/` 本地归档

## 1. 项目目标

把 QQ自由幻想端游的核心单人体验复刻为本地可玩的 iPhone 单机版本。体验目标不是社交 MMO，而是保留“建号 - 做任务 - 自动寻路 - 刷怪 - 掉装 - 强化 - 宠物 - 副本 - 长线养成”的爽感闭环。

## 2. 复刻原则

1. 玩法骨架高度对标：职业定位、成长曲线、刷怪节奏、装备养成、宠物养成、日常副本结构尽量贴近端游资料。
2. 单机化改造：剔除社交、玩家交易、组队依赖、经济市场、排行榜强竞争等多人系统。
3. 数据本地化：所有系统先沉淀为本地结构化数据，再进入开发实现，避免靠记忆或临时硬编码。
4. 美术 Q 版化：原图、地图、图标、怪物、装备外观先本地化为参考资料，再基于原版特征做 Q 版游戏资产；不把原版素材直接放进运行包。
5. 版本逐步扩容：P1 先做核心爽感竖切，P2 扩成端游主体系统，P3 进入完整对标，不再人为限制地图、宠物、副本、装备和养成内容数量。

### 3.2 结构化数据要求

每个可开发系统必须先产出本地数据表，建议放在 `data/design/`：

- `classes.json`：五职业基础定义、定位、武器偏好、成长系数。
- `skills.json`：主动技能、被动技能、技能等级、消耗、冷却、施法距离、伤害公式。
- `levels.json`：等级经验、属性点、技能点、开放系统。
- `attribute_formulas.json`：力量、智慧、体质、敏捷、精神等属性到战斗属性的换算。
- `maps.json`：主城、新手村、野外地图、副本地图、传送关系。
- `npcs.json`：NPC 坐标、功能、任务关系。
- `monsters.json`：怪物、精英怪、BOSS、等级、属性、技能、图鉴信息。
- `spawns.json`：刷怪点、刷新间隔、巡逻范围、精英/BOSS刷新规则。
- `quests.json`：主线、支线、职业、收集、循环、平行世界副本任务。
- `items.json`：消耗品、收集品、卡片、任务物品。
- `equipment.json`：武器、防具、饰品、职业外套、套装、外甲、基础属性。
- `drop_tables.json`：怪物、副本、活动掉落与权重。
- `enhancement_rules.json`：加工、打孔、镶嵌、升级、拆解、附魂、祝福。
- `pets.json`：宠物基础属性、成长类型、捕捉、喂养、信赖、经验。
- `pet_skills.json`：宠物技能、技能丸、技能池。
- `dungeons.json`：副本入口、层级、怪物波次、BOSS、奖励。
- `activities.json`：试炼、通天塔、怪物猎杀令等单机日常。
- `late_growth.json`：幻神、法宝、灵、战魂、典籍卡、称号、翅膀等后期养成。

### 3.3 字段溯源

关键数据字段必须带溯源字段：

- `source_url`：原始网页地址。
- `source_file`：本地归档 Markdown/HTML 路径。
- `source_confidence`：`official`、`wiki`、`player_observed`、`inferred`。
- `original_name`：端游原始名称。
- `project_name`：本项目可使用名称，允许原创化改名。
- `notes`：差异、推断或待验证说明。

## 4. 大版本规划

## P1：核心刷怪成长竖切版

### 4.1 版本目标

做出本地 iPhone 可玩的最小完整闭环：创建角色，选择五职业，跟随任务指引进入地图，自动寻路和自动战斗刷怪，获得经验、装备、宠物和副本奖励，形成 1-35 级左右的单机爽感。

### 4.2 玩家体验闭环

1. 创建角色并选择职业。
2. 在新手村接主线任务。
3. 自动寻路到目标 NPC 或怪物点。
4. 自动战斗刷怪，使用普通攻击和职业主动技能。
5. 自动补给药品，死亡后复活。
6. 掉落装备、消耗品、收集品。
7. 穿戴装备、修理耐久、学习技能、分配属性点。
8. 捕捉并召唤第一只宠物。
9. 进入 1-2 个单人副本，击杀 BOSS，获得阶段奖励。

### 4.3 功能范围

保留：

- 角色创建、五职业、等级成长、属性点。
- 技能学习/加点，P1 每职业先实现 4-6 个核心技能与 1-2 个被动。
- 自动寻路、自动战斗、自动补给、地图传送、任务指引。
- 普通攻击、主动技能、被动技能、职业定位。
- 野外刷怪、精英怪、BOSS战、单人副本、挂机刷怪、死亡复活。
- 主城、新手村、3-5 张野外地图、NPC、怪物图鉴。
- 主线任务、支线任务、收集任务、循环任务。
- 基础装备、武器、防具、饰品、消耗品、收集品。
- 装备基础属性、耐久/修理。
- 宠物捕捉、喂养、召唤、命名、宠物经验、宠物成长类型。

暂不做：

- 转职、飞升。
- 装备加工、打孔镶嵌、升级、拆解、附魂。
- 宠物技能丸、宠物培育、宠物装备、宠物法阵。
- 大量后期副本和后期养成。

### 4.4 P1 内容产出

地图内容：

- 1 个主城。
- 1 个新手村。
- 3-5 张野外地图。
- 1 个基础传送网络。
- 每张地图至少 3 类普通怪、1 类精英怪。

任务内容：

- 1 条 1-35 级主线。
- 每张地图 3-5 个支线任务。
- 5 条职业引导任务。
- 3 类循环任务模板：击杀、收集、跑腿。

战斗内容：

- 五职业基础职业定位。
- 每职业 4-6 个主动技能。
- 每职业 1-2 个被动技能。
- 普攻、命中、暴击、闪避、伤害浮字。
- 基础怪物 AI：近战追击、远程施法、BOSS 技能预警。

装备内容：

- 每职业 1-35 级武器、防具、饰品。
- 基础品质分层：普通、优秀、精良。
- 装备耐久和 NPC 修理。
- 怪物掉落与副本掉落。

宠物内容：

- 5-8 种可捕捉宠物。
- 3 类成长类型。
- 宠物召唤、跟随、参战、升级。

副本内容：

- 2 个单人副本：新手巢穴、初阶宝塔。
- 每个副本 2-3 段小怪加 1 个 BOSS。

### 4.6 P1 验收标准

- iPhone 上可离线进入游戏，无需账号服务器。
- 从创建角色到 35 级至少有 2 小时可玩内容。
- 自动寻路能完成主线任务中的 NPC 对话、地图移动、刷怪点到达。
- 自动战斗能连续挂机 20 分钟，包含补给、掉落拾取、背包满提示。
- 五职业都能完成 P1 主线，不出现职业卡死。
- 每个职业的战斗手感有差异，而不是只换技能名字。
- 所有 P1 核心数值都有本地数据来源和溯源字段。

### 4.7 P1.5 对标体验重制补充

P1.5 是 P1 的体验修正版，不新增 P2/P3 系统，重点解决“像网页样机、不像自由幻想”的问题。

已对齐：

- 原版资料优先：本地有资料先用本地；本地没有先补采，仍缺则记录到 `docs/prd/p1.5-source-gaps.md`。
- 五职业职业定位、武器、每级自动配点来自腾讯官方旧资料中心。
- P1 可玩技能子集改用官方技能名、等级和可确认倍率/效果；官方未给出的 MP、冷却、施法距离继续标为单机占位。
- 地图、怪物、任务优先使用官方旧资料中的桃源村、芳草林、龙城、粉粉兔、小蜗牛怪、食人花、野猪、树妖、鬼火等原名。
- 鬼火、小白兔、大蜗牛怪、山狼、火鸟官方参考图已本地化；游戏运行时使用 Q 版资产，不直接复用原图。
- iPhone 竖屏主界面改为游戏 HUD：角色血蓝、任务追踪、小地图、技能轮盘、底部入口、自动寻路/自动战斗。

仍不能算完全复刻：

- 属性公式、等级经验、怪物血攻防经验、低级装备数值仍有推断字段。
- 技能图标、完整 MP/CD/施法距离、技能升级曲线未补齐。
- 桃源村/芳草林/龙城地图原图、五职业原版男女形象、部分 P1 怪物原图仍缺。

### 4.8 P1.8 手游样式对标补充

P1.8 基于用户提供的横屏 MMO 手游截图，把首屏从竖屏 H5 面板改为横屏 iPhone PWA 单机手游体验。P1.8 必须遵守 `docs/prd/mobile-gameplay-standards.md`，未满足时只能标记为方向验证，不能标记为完成。

硬性标准：

- 横屏唯一体验；竖屏只提示“请横屏游玩”。
- 终态按 iPhone PWA 添加到桌面后的全屏体验验收。
- HUD 不堆叠：顶部入口数量受控，小地图、角色信息、任务栏、技能区不得互相遮挡。
- 单机原则：不设计伪玩家、好友、空间、排行榜竞争等 MMO 填充。
- 背包/地图/宠物/角色是游戏内浮层，打开后底层战斗继续，浮层不遮挡核心战斗区和右下技能区。
- 打怪必须可感知：选怪、靠近、攻击动作/反馈、飘字、死亡、掉落至少有基础表现。
- 角色、怪物、宠物必须进入 Q 版资产管线；符号人/临时几何图形只能标 `prototype_asset`。

验收方式：

- `npm run test:browser` 使用 844x390 横屏手机视口。
- 校验横屏唯一体验、无横向溢出、HUD 关键元素存在且不堆叠。
- 校验默认功能抽屉不遮挡游戏画面。
- 校验无伪玩家和社交入口。
- 校验战斗反馈与掉落反馈可见。
- 截图产物：`test-results/p1.8-landscape-smoke-thumb.png`。

## P2：端游主体系统扩展版

### 5.1 版本目标

把 P1 的“能刷”扩展为“像端游一样养”。重点补齐转职、装备加工、宠物深养、平行世界副本任务、单机日常和更多中期副本，让角色成长进入 35-80 级左右。

P2.0 的体验目标不是“功能存在”，而是达到个人可用的近商业手游基础体验：横屏战斗布局清楚，按钮不堆叠，UI 有图标、有质感、有清晰主次，交互不乱且表达明确。若战斗、任务、技能、属性、宠物、装备任一核心系统仍主要依赖文字按钮或线框表达，不能标记为 P2.0 完成。

### 5.2 玩家体验闭环

1. 完成转职任务，获得职业进阶技能。
2. 在中期地图刷怪、做循环任务、打日常副本。
3. 使用装备加工、打孔、镶嵌、升级和附魂提升战力。
4. 培养宠物技能、信赖度、成长类型和宠物装备。
5. 通过通天塔、试炼、怪物猎杀令等日常获得稳定资源。
6. 推进平行世界副本任务，解锁更多 BOSS 与装备。

### 5.2.1 P2 任务与留存节奏

P2 必须以自由幻想丰富任务体系为主参考，把前中期可玩性拆成两个阶段：

1. 前期 0-30 分钟靠任务牵引。玩家从登录、创建角色、主线、支线、职业引导、收集、循环任务和平行世界前置中自然学习移动、接怪、技能、加点、换装、宠物和副本入口，不能出现“打开游戏后不知道做什么”的空窗。
2. 约 30 分钟后进入玩法研究门槛。任务继续给目标，但推进效率开始依赖玩家理解属性加点、技能学习/加点、装备加工/镶嵌/附魂、宠物技能/信赖/培育、日常资源和副本机制。
3. 中期靠关卡、等级、副本和养成门槛牵引。玩家被转职、技能解锁、副本首通、通天塔层数、猎杀令目标、装备强化阶段和宠物成长目标持续拉住。
4. 所有游戏性、关卡节奏、任务牵引、数值门槛、玩家决策复杂度问题，优先找 CCGS/游戏策划视角评审。CCGS 结论与 PRD 冲突时，记录冲突并回到产品决策，不能静默折中。

### 5.6 P2 验收标准

- 从 35 级到 80 级至少有 8-12 小时可玩内容。
- P2 首 30 分钟必须可被任务体系连续牵引，覆盖主线、支线、职业、收集、循环或副本前置中的多种任务形态，并自然引导玩家接触技能、加点、装备、宠物和副本。
- 30 分钟后必须出现明确玩法研究空间：玩家需要通过加点、技能、装备、宠物、日常资源或副本策略做决策，才能提升推进效率。
- 五职业完成转职后战斗循环发生明显变化。
- 装备养成可以显著改变刷怪效率，但不会让主线数值崩坏。
- 宠物不是装饰，参战贡献可被玩家感知。
- 通天塔、试炼、怪物猎杀令能成为每日单机资源来源。
- 每个 P2 副本都有独立 BOSS 机制和奖励定位。
- 数据表支持热更新式调整，无需改代码才能调数值。
- 横屏 UI 和所有高频交互必须满足 `docs/prd/mobile-gameplay-standards.md`：按钮不堆叠，UI 有图标、有质感、有清晰主次，不乱、表达明确；战斗中角色、怪物、宠物、特效和伤害反馈不能层级错乱。

## P3：完整对标版

### 6.1 版本目标

以端游可考据资料为上限做完整对标，让游戏从“中期养成”进入“单机刷爽毕业”。P3 不再设置宠物、地图、副本、装备、活动数量上限，凡是资料能确认的端游单人可转化内容，都进入 P3 全量清单；资料暂缺的内容进入缺口表，不用缩减版替代。

### 6.2 玩家体验闭环

1. 完成飞升或高阶突破。
2. 解锁 80-110 级及端游资料中可考据的全部后期地图和高阶副本。
3. 刷高阶材料，推进幻神、法宝、灵、战魂、典籍卡、称号。
4. 追求装备祝福、龙翼秘宝、翅膀雕花等毕业属性。
5. 挑战高层通天塔和高难 BOSS。
6. 形成可长期离线/在线挂机的单机毕业路线。

### 6.4 P3 内容产出

地图内容：

- 补齐端游资料中可考据的全部后期地图，不设置数量上限。
- 所有后期野外地图都要沉淀地图名、入口、传送关系、怪物分布、精英/BOSS、任务关系和产出关系。
- 高阶地图产出与后期养成材料绑定；无法确认产出的地图标记为 `source_confidence=inferred` 或进入缺口表。

任务内容：

- 80-110 级主线。
- 飞升任务链。
- 后期系统解锁任务。
- 高阶副本前置任务。

装备内容：

宠物内容：

后期养成内容：

- 典籍卡：收集、激活、升级、组合属性。
- 称号：达成条件、属性、展示。

副本/活动内容：

- 端游资料可考据的全部高阶副本和活动，不限制数量。
- 通天魔劫塔。
- 高层通天塔。
- 高阶怪物猎杀令。

### 6.6 P3 验收标准

- 从 80 级到 110 级形成完整后期体验，时长由端游全量内容和数值节奏决定，不用固定小时数压缩范围。
- 毕业后保留端游资料可考据的全部可刷目标，包括装备、宠物、翅膀、幻神、法宝、灵、战魂、典籍卡、图鉴、称号和材料循环。
- 后期养成材料来源清晰，不依赖付费、交易或多人协作。
- 高阶副本有明确战力门槛、机制差异和掉落定位。
- 110 级后数值不出现单系统碾压全部系统的问题。
- 所有后期系统都有数据来源、推断标记和可调参数；资料缺失只能标记缺口，不能静默删减。

## 9. 技术落地建议

推荐先按 H5 实现，后续使用 iOS WebView 或轻量壳打包：

- 引擎：Phaser、PixiJS 或 Cocos Creator。
- 数据：本地 JSON，后续可转 SQLite。
- 存档：IndexedDB 或本地文件。
- 地图：Tilemap 数据驱动。
- 战斗：帧循环 + 技能配置驱动。
- AI：怪物行为和自动战斗都走配置优先级。
- 调试后台：本地 GM 面板，支持改等级、物品、金币、任务、传送、刷怪、掉落测试。

## 10. 风险与待补

1. 资料可信度：部分资料来自玩家百科，关键数值需要标记置信度，不能混成官方定论。
2. 资料缺口：高阶系统、后期副本、活动数值仍可能缺字段，需要在 `docs/reference/gaps.md` 继续维护。
3. 版权边界：名称、图片、地图布局、UI、音效不能直接复用；设计上需要保留机制体验，表达层原创化。
4. 个人工作室产能：P1/P2 必须克制范围，先验证战斗、任务、掉落、成长闭环；P3 作为完整对标版，范围不缩水，但可以分批开发和验收。
5. 数值膨胀：P3 系统很多，必须建立统一战力模型，否则后期会不可调。

## 11. 下一步拆解

优先顺序：

1. 建立 `data/design/` 数据表模板和字段规范。
2. 从 `docs/reference/catalog.json` 抽取 P1 所需职业、地图、怪物、装备、任务资料。
3. 做 P1 可玩原型：地图移动、自动寻路、战斗、掉落、背包、升级。
4. 做本地 GM 面板，支撑数值调试。
5. P1 验收后再进入 P2 系统扩容。
