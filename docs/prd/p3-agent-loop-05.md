# P3 多 Agent 本地化与试玩审计记录 Loop 05

更新时间：2026-07-05

## 本轮目标

在 Loop04 可进入高阶单机试炼的基础上，补齐最基本的副本机制感：复活池、限时、失败退出、战斗状态提示，并用真实浏览器进入通天魔劫塔验证。

## 本轮直接修复

- `enterDungeon()` 初始化副本机制状态：
  - `startedAt`
  - `timerSeconds/timeLeft`
  - `revivesLeft`
  - `waveIndex/remaining`
- 新增 `tickDungeonMechanics()`：
  - 每帧刷新限时。
  - 限时耗尽触发失败退出。
- `revive()` 改为副本优先：
  - 若副本有复活池，死亡后原地副本复活并继续刷当前波。
  - 复活池耗尽才失败退出。
  - 普通野外死亡仍回龙城。
- 新增 `failDungeon()`：
  - 清空战斗/副本状态。
  - 回到副本入口地图。
  - 恢复部分 HP/MP。
- `spawnDungeonEnemy()` 加空副本兜底，防止失败后旧延迟回调读空状态。
- 战斗状态栏显示当前试炼、波次、复活次数、剩余秒数。
- 后期副本卡展示试炼波数、复活池、限时和“本地平衡，非官方完整数值”。

## 仍需补齐

1. 高阶 BOSS 机制还只是限时/复活池，缺阶段技能、召唤、隐藏条件、评价奖励。
2. 高阶 BOSS 美术未 Q 版化，通天魔劫塔守关 BOSS 仍是 fallback 圆形。
3. 后期页还不是完整商业手游副本选择界面，缺今日次数、门槛状态、奖励预览、失败/通关记录。
4. 当前高阶副本数值为本地平衡模板，需要后续持续结合资料和测试调参。
